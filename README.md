## Задача 1
### Текст задачи
Обобщенная коробка.
Создайте сущность Коробка, которая обладает следующими характеристиками:
- Может хранить один произвольный объект в один момент времени.
- Объект можно получить и разместить на хранение в любой момент времени.
- Если объект забирают из коробки – ссылку на этот объект необходимо обнулить.
- Если объект кладут в коробку, но она не пуста – необходимо выкинуть исключение.
- Имеет метод проверки на заполненность.
- Методы класса должны работать с тем типом данных, который был указан во время создания объекта.
Создайте Коробку которая может хранить целочисленное значение, разместите туда число 3.
Передайте Коробку в какой-либо метод, извлеките значение, и выведите его на экран.

### Алгоритм решения
Класс ru.sivkova.box.GeneralizedBox<T> - это обобщенный (generic) класс, что позволяет ему работать с любым типом данных, указанным при создании экземпляра. Создано единственное поле obj  обобщенного типа T, что позволяет классу работать с любыми типами данных. Конструктор по умолчанию создает абсолютно пустую коробку, где поле obj инициализируется как null. Конструктор с параметрами позволяет создать коробку с уже размещенным внутри объектом, выполняет прямую инициализацию поля obj переданным значением. Конструктор копирования создает новую коробку как точную копию существующей, перед этим проверяя входной параметр на значение null. Геттер getObj() возвращает ссылку на объект obj без его удаления. Сеттер setObj(T obj) - метод для добавления объекта в коробку, вызывает busyBox() для проверки занятости. Если коробка не пуста, выбрасывается исключение, иначе объект сохраняется в поле obj. Метод take() позволяет взять объект из коробки. Сначала вызвает freeBox() для проверки наличия объекта. Если коробка пуста, выбрасывается исключение, иначе объект сохраняется во временную переменную, обнуляется поле obj и возвращается сохраненный объект. Метод isEmpty() - проверяет состояние коробки, возвращает true если поле obj содержит null, и false в противном случае. 
Переопределённый метод toString() выводит строковое представление состояния коробки. Также используются вспомогательные приватные методы. Метод busyBox() используется для проверки пред операцией добавления объекта. Если коробка не пуста (!isEmpty()), выбрасывает исключение с сообщением "Коробка уже занята!". Метод freeBox() используется для валидации перед операцией извлечения объекта. Если коробка пуста (isEmpty()), выбрасывает исключение с сообщением "Коробка пустая!".

Демонстрация работы класса в основной программе:
1. Создание экземпляра коробки - инициализируется пустая коробка, специализированная для работы с типом Integer, выводится сообщение для пользователя о создании;
2. Объявление переменной управления - int m для хранения выбора пользователя;
3. Вывод меню - отображаются все доступные операции с коробкой, чтение выбора пользователя;
4. Обработка команд (switch-case)
   - Команда 0: Завершение работы
   - Команда 1: Добавление объекта в коробку (приватный метод setBox())
   - Команда 2: Извлечение объекта из коробки (приватный метод takeBox())
   - Команда 3: Просмотр содержимого коробки (вызов toString())
  
Метод setBox(ru.sivkova.box.GeneralizedBox<Integer> box) запрашивает у пользователя ввод строки с числом, с помощью ru.sivkova.validator.Validator.valIsInt() проверяет корректность введённых данных, при корректном вооде вызывает box.setObj(value) с преобразованным числом, иначе выбрасывает исключение с соответствующим сообщением.

Метод takeBox(ru.sivkova.box.GeneralizedBox<Integer> box) вызывает box.take() и выводит сообщение об извлечении объекта из коробки.

В классе ru.sivkova.validator.Validator реализован метод valIsInt, который проверяет строковые значения на соответствие целочисленному формату. Сначала проверяет, что переданная ссылка не является null, далее выполняет двойную проверку:trim() - удаляет пробелы, а isEmpty() - проверяет, что после trim строка не пустая. parseInt() - статический метод класса Штеупук пробует преобразовать строку в целое число.

Программа обрабатывает следующие исключения:
1. IllegalStateException - при попытке добавить в занятую или извлечь из пустой коробки;
2. NumberFormatException - при невалидном вводе числа;
3. IllegalArgumentException - при некорректных параметрах.

## Задача 3
### Текст задачи
Сравнимое.
Создайте ссылочный тип Сравнимое, гарантирующий наличие по данной ссылке метода со
следующими характеристиками:
- Называется “сравнить”
- Принимает объект.
- Тип принимаемого объекта может быть изменен без изменения самого Сравнимого.
- Возвращает целое число.

### Алгоритм решения
Обобщённый интерфейс ru.sivkova.comparable.MyComparable<T> предоставляет способ сравнения объектов различных типов. Параметризация типом T - позволяет работать с любыми типами данных. Метод compare - гарантирует наличие логики сравнения. Возвращается целочисленный результат: 1 (больше), -1 (меньше), 0 (равно). Для демонстрации созданы классы ru.sivkova.comparable.ComparableNumber для реализации числовых типов и ru.sivkova.comparable.ComparableString для реализации строковых типов. Оба класса используют наследование - implements ru.sivkova.comparable.MyComparable<String> и содержат методы compare(), реализующие алгоритм сравнения.

Демонстрация работы класса в основной программе:
1. Пользователь выбирает одну из доступных операций сравнения;
2. Обработка выбора через switch-case
   - Команда 0: Завершение работы
   - Команда 1: Сравнение чисел (Ввод и валидация двух чисел любых типов, создание объектов ru.sivkova.comparable.ComparableNumber, выполнение сравнения, вывод результата)
   - Команда 2: Сравнение строк (Ввод двух строк, создание объектов ru.sivkova.comparable.ComparableString, сравнение двух созданных объектов, вывод результата).
  
Для проверки корректности введённых для сравнения чисел в классе ru.sivkova.validator.Validator реализованы методы valIsInt(String value) - проверка строки на соответствие целочисленному формату, valIsDouble(String value) - проверка строки на соответствие формату чисел с плавающей точкой, valIsNumber(String value) - проверка на числовой формат (целый или дробный).

Приватный метод parseNumber(String str) принимает строку, определяет тип числа, содержащегося в строке и возвращает число соответствующего типа (int или double).

Метод resultCompare(int result) преобразовывает числовой код в символ:
- 1 → ">" (больше)
- -1 → "<" (меньше)
- 0 → "=" (равно)

## Задача 2
### Текст задачи
Поиск максимума.
Создайте метод, принимающий набор Коробок из задачи 3.1.1 и возвращающий максимальное из их значений в формате double. Принимаемые методом Коробки могут быть параметризованы любыми видами чисел.

### Алгоритм решения
Создан класс ru.sivkova.box.Maximum для поиска максимального значения среди всех коробок. Класс содержит единственный метод findMax(), который в качестве параметра принимает коллекцию List, состоящую из объектов класса ru.sivkova.box.GeneralizedBox (объекты класса могут быть параметризованы любым типом, который является наследником Number). В методе сначала производится проверка на null ссылку и на пустую колекцию, елс проверка не пройдена выбрасывается исключение. При прохождении проверки инициализируются пременные max (устанавливается в минимально возможное значение double), find (логическая переменная, указывающая на наличие хотя бы одного корректного значения). Далее производится последовательный обход всех коробок в списке, пустые коробоки пропускаются с помощью проверки isEmpty(). Из каждой коробки извлекается числовое значение с помощью метода getObj(), преобразовывается в double через doubleValue() для сравнения, при нахождении большего значения переменная max обновляется. Также осуществляется валидация, что был найден хотя бы один числовой элемент (если максимум не найден, выбрасывается исключение с сообщением об ошибке, что все коробки пустые). Максимальное значение возвращается в формате double. 

Демонстрация работы класса в основной программе:
1. Инициализация динамической коллекции для хранения созданных коробок;
2. Пользователь выбирает одну из предложенных операций;
3. Обработка выбора через switch-case
    - Команда 0: Завершение работы
    - Команда 1: Просмотр коробок (Производится проверка наличия коробок, последовательный вывод каждой из существующих коробок с помощью метода toString() из класса ru.sivkova.box.GeneralizedBox.)
    - Команда 2: Создание коробки с числом (Реализован бесконечный цикл до получения кооректного ввода, проверка осуществляется с помощью метода valIsNumber() из класса ru.sivkova.validator.Validator, принеобходимости выводится соответствующее сообщение о некорректном вводе. При вводе коректного значения определяется тип введённого числа с помощью методов valIsInt() и  valIsDouble() из класса ru.sivkova.validator.Validator и создаётся коробка соответствующего типа, которая добавляется в созданную коллекцию. Выводится сообщение об успешном создании.)
    - Команда 3: Поиск максимального значения (Проверяется наличие коробок в коллекции. Далее выполняется поиск максимума с помощью метода findMax(), обрабатываются возможные исключения, результат выводится пользователю.)
  
Для проверки корректности ввода в классе ru.sivkova.validator.Validator реализованы методы valIsInt(String value) - проверка строки на соответствие целочисленному формату, valIsDouble(String value) - проверка строки на соответствие формату чисел с плавающей точкой, valIsNumber(String value) - проверка на числовой формат (целый или дробный).
